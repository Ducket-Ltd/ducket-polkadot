**1. Create `frontend/src/hooks/useEventData.ts`:**

This hook fetches all 6 events and their 12 tiers in batched multicall using wagmi v2 `useReadContracts`, then merges with static metadata.

```typescript
import { useReadContracts } from 'wagmi'
import { CONTRACT_ADDRESS, DUCKET_ABI } from '@/lib/contract'
import { EVENT_METADATA, type EventMeta } from '@/data/eventMetadata'
import { useMemo } from 'react'
```

Export `useEventData()` function:

1. Derive `eventIds = Object.keys(EVENT_METADATA).map(Number)` — should be [1,2,3,4,5,6]
2. For each eventId, derive its tier tokenIds from `Object.keys(EVENT_METADATA[eventId].tiers).map(Number)`
3. Build `contracts` array for `useReadContracts`:
   - First: one `getEvent` call per eventId (6 calls)
   - Then: one `getTicketTier` call per tokenId (12 calls)
   - Total: 18 calls batched into one multicall
4. Call `useReadContracts({ contracts })` — returns `{ data, isLoading, isError, refetch }`
5. In a `useMemo`, process `data`:
   - Split results: first 6 are event results, next 12 are tier results
   - For each event result where `status === 'success'` and `result.exists === true`:
     - Merge on-chain fields (eventDate as `new Date(Number(result.eventDate) * 1000)`, maxResalePercentage, totalSupply, maxTicketsPerWallet, resaleEnabled, transferEnabled, organizer) with static metadata from `EVENT_METADATA[eventId]`
     - For each tier tokenId belonging to this event, find the corresponding tier result and merge on-chain fields (price, stablePrice, maxSupply, minted, exists) with tier metadata (name, description from EVENT_METADATA)
   - Return array of merged event objects, each containing a `tiers` array
6. Return `{ events, isLoading, isError, refetch }`

Export a `MergedEvent` type and `MergedTier` type for downstream consumption:
```typescript
export interface MergedTier {
  tokenId: number
  tierName: string
  description: string
  price: bigint        // DOT price in wei (18 decimals)
  stablePrice: bigint  // USDC price (6 decimals)
  maxSupply: bigint
  minted: bigint
  exists: boolean
}

export interface MergedEvent {
  eventId: number
  // On-chain
  eventDate: Date
  maxResalePercentage: number
  totalSupply: bigint
  maxTicketsPerWallet: bigint
  resaleEnabled: boolean
  transferEnabled: boolean
  organizer: string
  // Metadata
  name: string
  description: string
  imageUrl: string
  venue: string
  city: string
  country: string
  category: string
  // Merged tiers
  tiers: MergedTier[]
}
```

CRITICAL wagmi v2 notes:
- Each `data[i]` is `{ status: 'success' | 'failure', result }` — always check `status === 'success'` before accessing `result`
- Contract args must be bigint: `args: [BigInt(eventId)]`
- `address` must be cast: `address: CONTRACT_ADDRESS as \`0x${string}\``

Export `useMyTickets()` function:

1. Get `address` from `useAccount()`
2. Build `contracts` array: one `balanceOf` call per tokenId in `ALL_TIER_TOKEN_IDS` (12 calls), with `args: [address!, BigInt(tokenId)]`
3. Call `useReadContracts({ contracts, query: { enabled: !!address } })` — disabled when no wallet connected
4. In a `useMemo`, process results:

     ```typescript
       eventImageUrl: string

     }
     ```
   - Group owned tiers by eventId, so each event appears once with its owned tiers underneath
5. Return `{ ownedByEvent: OwnedTicketGroup[], isLoading, refetch }`
